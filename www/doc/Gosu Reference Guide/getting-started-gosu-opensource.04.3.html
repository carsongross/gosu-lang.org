Getting Started with Gosu in Eclipse
Using Gosu Tester to Write ‘Hello World’
The easiest thing to confirm that Gosu is installed in Eclipse is to use the Gosu Tester window.
To open the Gosu Tester window
1.
Click the menu item Window → Show View → Gosu Tester
Note: If the Gosu Tester item does not appear, you did not perform all the steps in installation. Pay particular note to switching to the Java perspective in Eclipse and then performing the Reset Perspective action. See "Installing the Eclipse Gosu Editor Plugin" for details.
2.
If the Gosu Tester pane appears in an undesired location, drag its title bar to the desired location.
3.
The typing focus is in the Gosu Tester pane in Eclipse.
You can type Gosu code directly into the Gosu Tester pane and click the Execute button to run the code.
For example, type the following simple Gosu program:
print("Hello World!")
Next, click the Execute button to runs your program.
In the pane to the right, which is white text on black background, you can see the output. For the previous example, the output is:
Hello World!
Congratulations! You just ran your first Gosu program.
Creating a Gosu Project in Eclipse
To create a new Gosu project and a new Gosu program
1.
Click the menu item File → New → Gosu Project. A Gosu project dialog appears.
Note: If the Gosu Project item does not appear, you likely did not perform all installation steps. Pay particular note to switching to the Java perspective in Eclipse. If necessary, repeat the Reset Perspective action. See "Installing the Eclipse Gosu Editor Plugin" for details.
IMPORTANT In Eclipse, if any files contain Gosu, the project type must be Gosu Project. Your Gosu project can contain other Java files and other resources. The initial program that you run must be a Gosu program, which is a .gsp file. Your Gosu program can use other Gosu classes, Java classes, and other Gosu types. However, if the types you use are not built-in to Gosu or Java, you must add these resources to your Gosu project. For more information about Gosu programs, see "The Structure of a Gosu Program".
2.
In the Project Name field, type a project name, such as GSProjectSimple.
3.
Modify any other optional fields in that dialog.
4.
Click Finish. Eclipse may take a while to set up your new Gosu project.
5.
Your new Gosu project appears in the Package Explorer pane in Eclipse.
6.
Create a package (namespace hierarchy) to contain your Gosu program. Right click on the src folder underneath your project folder icon. Right-click on the src folder and choose New → Package. In the next dialog, in the Name field, type the word example. There is now a top-level package called example.
7.
Right-click on the icon for the example package. From the contextual menu, choose New → Gosu Program. This action tells Eclipse to create a new program in the example package.
Note: If the Gosu Program item does not appear, you likely did not perform all installation steps. Pay particular note to switching to the Java perspective in Eclipse. If necessary, repeat the Reset Perspective action. See "Installing the Eclipse Gosu Editor Plugin" for details.
8.
In the dialog that appears, in the Program Name field, type a program name that starts with a capital letter. Do not add a file extension (such as .gsp) in this dialog because Eclipse interprets it as a package qualification.
9.
If you want to change which package for your program, next to the Package field, click Browse and select a new package.
10.
Click Finish. Eclipse creates your new Gosu program.
11.
In the Package Explorer pane, navigate to the following path and click on the item:
project_name → src → example → program_name
12.
In the Gosu editor for your Gosu program file, type or paste the following Gosu code:
print("Hello from Gosu!")
13.
Click the menu item Run → Run.
14.
The Console window appears and displays the text:
print("Hello from Gosu!")
15.
Congratulations! You just created a Gosu project and created a file-based Gosu program!
IMPORTANT You can run Gosu programs (.gsp files) from within Eclipse. You cannot run a Gosu class file or other types file directly from the Eclipse user interface. If you want to call a Gosu class (or other type of file), make a simple Gosu program that uses your other types. For example, instead of defining a main() method in a class and telling the IDE your main class, your .gsp file is your main file. Your main .gsp file can call static methods on your main class with syntax: myClass.main(). Or create an instance of your class with syntax new myClass().main().
The types of Gosu files include:
•
Gosu programs - provide a way to write small scripts or to launch larger Gosu applications
•
Gosu classes - the core structure for encapsulating data and functions
•
Gosu enhancements - add properties methods to existing types
•
Gosu interfaces - a set of methods that other types (implementing classes) must implement
•
Gosu enumerations - an encapsulated set of unique identifiers associated with unique numbers
•
Gosu templates - provide a convenient way to build large text objects
Running and Debugging Gosu in Eclipse
To run and debug Gosu in Eclipse, you can use the standard Eclipse tools, panes, and windows:
•
if you run a Gosu program, it does not stop at breakpoints
•
double click to the left of a line to set or a clear a breakpoint
•
if you run a Gosu program in debug mode (the menu items says Debug), Gosu stops at breakpoints. When stopped, you can examine variables or step over, step in, or step out in the Gosu editor. This is the same flow as with other languages such as Java within Eclipse.
•
open the console window to see Gosu output of the print function, or exceptions during execution.
•
open the variables panel to view the current state of variables
In addition, the Gosu Tester window lets you run short bits of code and see the output quickly. The Gosu Tester window has access to Gosu and Java classes in your Eclipse project. However, you cannot use the Eclipse debugger from within the Gosu Tester window (you cannot set breakpoints there).
Gosu Editor Features
The Gosu editor includes the following features:
•
Quickfix. The Gosu editor offers suggestions to fix code, insert uses statements, add a cast operation, add interface requirement methods, or fix capitalization. Type ALT-RETURN to accept the suggestion.
•
Dot completion. After you type a period character, Gosu shows a list of valid objects after the period (see "Using Dot Completion"). This feature uses the advanced Gosu type system to ensure that only valid (and non-deprecated) properties and methods appear.
•
Code autocompletion. Press CNTL-SPACE to see suggestions for completing the current code. For example, if there is a method call and you typed none of the name or part of the name, code autocompletion offers suggestions based on what you already typed.
•
Error highlighting. The Gosu editor highlights errors automatically
•
Find original definitions. CNTL-click on a term in Gosu code to jump to where that object is defined, even in Java code
•
Mark occurrences. Automatically mark all occurrences of what you select or currently place the cursor. The editor only marks that term in that file, and only within the current code context. This means that if two different functions have a local variable of the same name, Gosu considers them different. The mark occurrences feature works with most Gosu files but does not work with Gosu templates.
All keyboard shortcuts use the Eclipse defaults in the same as Java. Refer to the Eclipse documentation for details.
Using Dot Completion
You can use the Gosu editor to complete your code by placing the cursor at the end of a valid Gosu object or subobject. Next, type a period (dot) character. This action causes the editor to open a pop-up window that contains all the subobjects and methods that are valid for this object in this context.
As you type, Gosu filters this list to include only those choices that match what you have typed thus far. Use the down arrow to highlight the item you want and press Enter, the space bar, or Tab to select it. After you select an item, Gosu inserts it in the correct location in the code.
Dot completion also works with Gosu or Java packages. You can enter a package name and press dot to get a list of packages and types within the package name before the dot. The Gosu editor can complete all packages and namespaces within the Gosu type system including. Gosu filters the list as you type to include only those choices that match what you have typed thus far.
Other Eclipse Features
If you use the Eclipse search feature, it searches through Gosu classes.
Note: This only works for classes, not for Gosu Programs, enhancements, or templates.
This release does not support Eclipse refactoring tools.

In the Gosu pane:
•
Syntax coloring customization - For each type of element in the language, customize how to show them. For example, specific colors for class names.
In the General pane:
•
To set the font choices: go the pane Preferences → General → Appearance → Colors and Fonts → Gosu
Note: If your system has the font Consolas, that is a good choice for Gosu coding.
In the Java pane:
•
Syntax coloring
