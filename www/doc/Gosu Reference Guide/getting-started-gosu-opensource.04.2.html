Installing the Eclipse Gosu Editor Plugin
The Gosu community release includes an Eclipse editor plugin for Gosu. The editor includes text editing, intelligent auto-complete, automatic compilation error detection, debugging support, and other common Eclipse features.
To install the Gosu Eclipse plugin
1.
From the Eclipse web site, download Eclipse 3.5. You must use one of the package that includes Java Development Tools (JDT). For example, download the install called “Eclipse IDE for Java Developers”
http://www.eclipse.org/downloads/packages/release/galileo/sr2/
IMPORTANT You must use Eclipse version 3.5, which is also called Galileo. Do not use version 3.6, which is also called Helios.
2.
Start Eclipse.
3.
Click the menu item Help → Install new software...
4.
In the dialog that appears, navigate to the Work with field and paste in the following URL:
http://gosu-lang.org/eclipse
5.
Press the Enter key. Eclipse gets information from the software install web site and updates the main pane in the window.
6.
In the main pane of that window, click the checkbox next to the item Gosu Eclipse Tools to mark it for download.
7.
Click Next >. Eclipse displays a progress bar labelled “Calculating requirements and dependencies.” This might take a while, depending on the speed of your network connection and your computer. When it is done, Eclipse displays the Install Details dialog.
8.
In the Install Details dialog, click Next > again to confirm
9.
Read the license agreement
10.
Click I accept the terms of the license agreements
11.
If the button labelled Finish is enabled, click it to begin installation.
If the Finish button is disabled (you cannot click it), this is an Eclipse bug. To workaround this, go back two steps in the installation wizard to the first step in the install software wizard, then repeat the instructions at step 4. In other words, click the Back button twice, then repeat the last two steps until the Finish button on this screen is enabled so you can click it.
12.
During installation, Eclipse may display a dialog to warn you that you are installing unsigned content. Click OK in that dialog if it appears.
Similarly, Eclipse may ask you to confirm that you trust digital signature certificates that protect signed content from the Eclipse.org Foundation. You must check the checkbox next to the certificates to select them. Only after selecting the certificates by checking the checkboxes, click OK in that dialog.
WARNING You must check the checkbox next to any certificates in that dialog. If you do not, Gosu installation fails.
13.
Eclipse prompts you to restart the Eclipse application. Click Yes to let Eclipse restart itself.
14.
Wait for Eclipse to restart.
15.
Open the preferences window.
•
For most Eclipse versions, click the menu item Window → Preferences.
•
On the Mac OS X platform, click Eclipse → Preferences.
16.
In the left list of preferences, click JDT Weaving. The dialog looks similar to the following:
17.
Find the text above the button that begins JDT Weaving is currently... If the next word is ENABLED, JDT weaving is already enabled. This would only be the case if you did not use a fresh installation of Eclipse. If the next word is ENABLED, skip to step 19.
18.
If you got to this step, the text says JDT Weaving is currently DISABLED. Perform the following steps:
a.
Click the button underneath that says Click to ENABLE.
b.
Eclipse asks you to confirm. Click Yes to confirm.
c.
Eclipse prompt you to restart. Click Yes to restart Eclipse.
d.
Wait for Eclipse to restart again.
e.
Open the preferences window again and navigate to the JDT Weaving preferences again.
f.
Confirm that the text now says JDT Weaving is currently ENABLED.
g.
If the text instead says JDT Weaving is currently DISABLED, enabling JDT Weaving failed. This might be due to one of the installed Eclipse plugins throwing an exception during startup. This might be other third-party plugins, or might be the Gosu plugin. Go to the menu item Window → Show View → Error Log. Select all lines in the Error Log pane. Look in the Plug-in column for any errors. If there are any errors for the Gosu plugin, right-click the error and select Copy. Finally, please send that error report to the Gosu team with a description of what happened, and details of your Eclipse and computer configuration.
Depending on the cause of the problem, you might be able to manually enable JDT Weaving using the following instructions
    http://wiki.eclipse.org/JDT_weaving_features#Manual_Install
19.
Switch to the Java perspective in Eclipse. Click the menu item Window → Open Perspective → Java. Do not select the menu item Java Browsing by accident, as that is a different Eclipse Perspective. If the menu does not contain the menu item that simply says Java, then select Other... and then in the following dialog choose Java (default).
Note: Gosu is built on top of the Java JVM. The Gosu editor does not have its own Eclipse perspective. Instead inherits from the Java perspective.
20.
Click the menu item Window → Reset Perspective.... This adds the Gosu-specific menu items to the File → New... submenu and the contextual menu version of this menu.
WARNING This step is strongly recommended because it improves greatly usability within Eclipse. However, if you customized the pane positions within Eclipse for the Java perspective, be aware these settings revert to the default locations.
21.
Confirm the plugin is installed with the correct Eclipse perspective settings. Click the menu item File → New. Confirm that the following menu item exists: File → New → Gosu Class. If it does not appear, carefully review the instructions again and repeat the installation. Notes:
•
These menu items appear even if you did not successfully enable JDT Weaving. However, JDT Weaving is required to actually use the Gosu editor. See step 18.
•
These menu items appear only when you are in the Java perspective in Eclipse. See step 19.
22.
Begin to use the Gosu Tester and create Gosu projects in Eclipse. For details, see "Getting Started with Gosu in Eclipse".
